About the Model

Architecture, training data, feature engineering, and performance metrics

Model Overview

The predictor uses a Gradient Boosting Regressor (scikit-learn) selected automatically after comparing Linear Regression, Random Forest, Gradient Boosting, and XGBoost by held-out R² score.

All categorical fields (house type, location, city) are label-encoded using LabelEncoder objects persisted alongside the model. Numeric features are passed directly; no scaling is applied for tree-based methods.

Prediction Pipeline
1

Input Validation

Form fields are validated server-side. Numeric fields are cast to int/float; empty or invalid values return an error message.

2

Label Encoding

House type and location strings are transformed to integers using the same LabelEncoder objects used during training (models/encoders.pkl).

3

Feature Assembly

A named pandas.DataFrame is constructed with the exact column order the model was trained on (read from feature_info.json).

4

Gradient Boosting Inference

The loaded model calls .predict(X). The raw output (£) is rounded and returned.

5

Confidence Range

A ±10% band is applied to the prediction to communicate model uncertainty to the user.

Strengths
  • Trained on real London property transaction data
  • Best R² among 4 algorithms evaluated
  • Handles non-linear feature interactions
  • Named DataFrame prevents column-order bugs
  • Encoders persisted separately for reproducibility
  • Sub-second inference
Limitations
  • R² of 0.66 — significant variance remains unexplained
  • No temporal features (market cycle, year of sale)
  • Property condition and interior quality not captured
  • City/County defaulted to index 0 in current form
  • Unseen locations fall back to index 0 encoding
  • Use as guidance only, not a formal valuation
